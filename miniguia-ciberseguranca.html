Contexto & Objetivos

Em um mundo onde dados valem mais do que petróleo, a cibersegurança deixou de ser exclusividade de especialistas. Este caderno foca no nível intermediário — com base em redes e sistemas — avançando para modelagem de ameaças, criptografia aplicada e frameworks de defesa.

Objetivos de Estudo
  • Criptografia simétrica, assimétrica e hashing
  • Kill Chain & MITRE ATT&CK
  • Modelagem de ameaças com STRIDE
  • OWASP Top 10 (2021)
  • NIST Cybersecurity Framework 2.0
  • Ferramentas de red/blue team
Ferramentas do Blue Team
  • Splunk / ELK Stack (SIEM)
  • Snort / Suricata (IDS/IPS)
  • Wireshark (análise de tráfego)
  • OpenVAS / Nessus (scanner)

  • Metasploit Framework
  • Burp Suite (AppSec)
  • Nmap (reconhecimento)
  • Hashcat / John the Ripper

// 04 · MINIGUIA · PARTE 1

A Tríade CIA

Os três pilares fundamentais da Segurança da Informação — a base sobre a qual todos os controles, frameworks e práticas são construídos.

C
Confidencialidade
Acesso apenas por quem tem autorização formal. Dados visíveis somente para os destinatários corretos.
⚠ Interceptação · Credential theft · Eavesdropping
🛡 Criptografia · MFA · Controle de acesso
I
Integridade
Dados não alterados de forma não autorizada durante armazenamento, transmissão ou processamento.
⚠ Tampering · SQL Injection · MITM
🛡 SHA-256 · Assinaturas digitais · Checksums
A
Disponibilidade
Sistemas e dados acessíveis quando necessários. Falhas não podem ser aproveitadas como vetor de negação de serviço.
⚠ DDoS · Ransomware · Falhas de hardware
🛡 Redundância · Backups · Rate limiting
Criptografia Aplicada
TipoExemploChavesVelocidadeCaso de Uso Ideal
Simétrica AES-256 1 chave (cifrar/decifrar) Rápida Disco, VPNs, bancos de dados, armazenamento em repouso
Assimétrica RSA-4096, ECC Par pública/privada Lenta TLS, SSH, assinaturas digitais, PKI, troca de chaves
Hashing SHA-256, bcrypt Sem chave (mão única) Muito rápida Senhas, integridade de arquivos, assinaturas digitais
TLS (híbrido) TLS 1.3 Handshake assimétrico + sessão simétrica Otimizada HTTPS, comunicação segura em rede — o melhor dos dois mundos

Glossário Técnico

20 termos identificados pelo NotebookLM como os mais recorrentes nas 5 fontes carregadas.

glossary.db — 20 entries
$ query glossary --all --format=table
APT
Advanced Persistent Threat — ator sofisticado (geralmente estado-nação) com presença prolongada em redes-alvo sem ser detectado.
→ MITRE ATT&CK
Attack Surface
Conjunto de pontos de entrada (portas, APIs, endpoints, usuários) que podem ser explorados por um atacante.
→ Bishop — Computer Security
Blue Team
Time responsável pela defesa ativa de sistemas — monitoramento contínuo, detecção de ameaças e resposta a incidentes.
→ NIST CSF
CVSS
Common Vulnerability Scoring System — sistema padronizado de pontuação de severidade de vulnerabilidades em escala 0.0 a 10.0.
→ OWASP
Defense in Depth
Estratégia de múltiplas camadas de controles de segurança — se uma camada falha, as demais compensam e limitam o dano.
→ Bishop + NIST CSF
Exploit
Código, script ou técnica que aproveita uma vulnerabilidade específica para executar ações não autorizadas em um sistema.
→ MITRE ATT&CK
Hardening
Processo de reduzir a superfície de ataque removendo serviços desnecessários, fechando portas e aplicando configurações seguras.
→ NIST CSF
IOC
Indicator of Compromise — artefato forense (hash, IP malicioso, domínio, string) que indica que um sistema foi comprometido.
→ MITRE ATT&CK
Least Privilege
Princípio de conceder apenas as permissões mínimas necessárias para cada usuário ou processo executar sua função específica.
→ Bishop — Computer Security
MITM
Man-in-the-Middle Attack — atacante se posiciona entre duas partes comunicantes para interceptar, modificar ou injetar dados.
→ Stuttard & Pinto
Payload
Parte do código malicioso que executa a ação danosa (exfiltração, cifragem, destruição), diferente do vetor de entrega.
→ MITRE ATT&CK
Pentesting
Teste de penetração — simulação controlada e autorizada de ataques para identificar vulnerabilidades antes que atacantes reais o façam.
→ Stuttard & Pinto
Red Team
Time que simula adversários reais de forma persistente para testar a eficácia dos controles e da detecção de uma organização.
→ MITRE ATT&CK
SIEM
Security Information and Event Management — plataforma que centraliza, correlaciona e analisa logs de múltiplas fontes em tempo real.
→ NIST CSF
Threat Modeling
Processo estruturado de identificar ameaças, vulnerabilidades e contramedidas durante o design de um sistema, antes da implementação.
→ Bishop — Computer Security
TTP
Tactics, Techniques, and Procedures — descrição detalhada do comportamento observado em atacantes, catalogado no MITRE ATT&CK.
→ MITRE ATT&CK
XSS
Cross-Site Scripting — injeção de scripts maliciosos em páginas web que são executados no navegador de outros usuários.
→ OWASP A03
CSRF
Cross-Site Request Forgery — força o navegador de uma vítima autenticada a executar ações não intencionadas em um site confiável.
→ OWASP
Pivoting
Usar um sistema já comprometido como trampolim para atacar outros sistemas na rede interna, expandindo o acesso lateral.
→ MITRE ATT&CK
Zero-Day
Vulnerabilidade desconhecida pelo fabricante do software, para a qual ainda não existe patch disponível — altamente valorizada por atacantes.
→ OWASP + MITRE

Prompts Reutilizáveis

6 prompts testados e validados para apoiar futuras sessões de estudo no NotebookLM ou qualquer LLM.

// ANÁLISE DE VULNERABILIDADE
Com base nos documentos carregados, analise a vulnerabilidade [NOME/CVE/CATEGORIA] nos seguintes aspectos: 1. Mecanismo técnico de funcionamento 2. Vetor de ataque típico (passo a passo simplificado) 3. Sistemas/configurações afetados 4. Contramedidas preventivas e corretivas 5. Referência MITRE ATT&CK (tática + técnica ID) se aplicável
// MODELAGEM DE AMEAÇAS (STRIDE)
Aplique o framework STRIDE para modelar as ameaças de [SISTEMA/COMPONENTE]. Para cada categoria (Spoofing, Tampering, Repudiation, Information Disclosure, Denial of Service, Elevation of Privilege): - Descreva a ameaça específica para este contexto - Cite um ataque concreto possível - Proponha uma contramedida técnica implementável
// MAPEAMENTO MITRE ATT&CK
Com base no documento MITRE ATT&CK carregado, trace o caminho de um ataque de [TIPO: ransomware / APT / insider threat / supply chain]. Para cada tática utilizada: - Identifique a tática (TA####) e nome - Cite a técnica mais provável (T#### com nome) - Descreva como um blue team poderia detectar essa atividade
// REVISÃO DE CONTROLES (NIST CSF)
Com base no NIST CSF e nos demais documentos carregados, avalie os controles de segurança para [TIPO DE SISTEMA: aplicação web / API REST / cloud]. Para cada função do NIST (Govern, Identify, Protect, Detect, Respond, Recover): - Identifique o controle mais crítico - Descreva como implementá-lo - Indique o risco mitigado
// ANÁLISE DE INCIDENTE
Com base nos documentos carregados, analise o seguinte cenário de incidente: [DESCREVA O CENÁRIO] Responda: 1. Quais categorias do OWASP Top 10 foram potencialmente exploradas? 2. Qual sequência de táticas MITRE ATT&CK melhor descreve o ataque? 3. Quais controles do NIST CSF falharam? 4. Que ações de resposta e recuperação deveriam ser tomadas?
// AUTOAVALIAÇÃO TÉCNICA
Com base em todo o material carregado sobre [SUBTÓPICO DE SEGURANÇA], crie 8 questões de nível intermediário-avançado: - 3 questões conceituais (4 alternativas + gabarito comentado) - 3 questões de aplicação prática (cenário + solução esperada) - 2 questões "pegadinha" baseadas em erros comuns de profissionais
